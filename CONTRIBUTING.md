# Contributing to ProofMoney Docs

ProofMoney welcomes contributions related to:

- protocol documentation;
- proof model design;
- software architecture;
- security review;
- wallet safety;
- user education;
- risk disclosure;
- developer tooling;
- public verification interfaces.

Before contributing, please read:

- `docs/00-project-principles-v1.0.md`
- `docs/06-risk-disclosure-v1.0.md`
- `docs/07-forbidden-activities-v1.0.md`

Contributions that promote speculation, private allocation, misleading financial expectations, or unsafe user behavior will not be accepted.

## Contribution Boundaries

Do not submit content that includes:

- PRM sale language;
- yield promises;
- price predictions;
- private allocation language;
- test-unit value claims;
- exchange listing promises;
- misleading financial promotion.

ProofMoney is a verification-first protocol initiative.

If money cannot be verified, it is only a promise.
